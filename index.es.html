Características

Mercury ofrece comunicaciones de datos HF de alta calidad en un paquete ligero y open-source.

Protocolo ARQ Adaptativo

Corrección inteligente de errores con handshake connect/accept, lógica ACK/reintento, keepalive y desconexión controlada. Cada byte entregado, incluso en condiciones adversas.

Fiabilidad

Cambio Inteligente de Modulación

Selección adaptativa de modo por dirección: cambia automáticamente entre DATAC4, DATAC3 y DATAC1 según la retroalimentación SNR en tiempo real. Rendimiento máximo, siempre.

Rendimiento

Modo Broadcast

Envía datos simultáneamente a múltiples estaciones en paralelo junto con sesiones ARQ. Tramas de broadcast dedicadas con su propio puerto TCP de entrada — un transmisor, muchos receptores.

Escalabilidad

Compatible con VARA TCP

Reemplazo directo para VARA TNC. La misma interfaz de socket de control y datos significa que Mercury funciona instantáneamente con Winlink, JS8Call y todas las aplicaciones de radioafición existentes.

Compatibilidad

Multiplataforma

Binarios nativos para Linux (AMD64 y ARM64 / Raspberry Pi) y Windows. Backends de audio ALSA, PulseAudio, WASAPI. Funciona en cualquier hardware con tarjeta de sonido.

Portabilidad

Independiente de Radio vía HAMLIB

Controla miles de transceptores HF mediante HAMLIB — Icom, Yaesu, Kenwood y más. Activación PTT automática. Funciona con prácticamente cualquier radio HF de aficionado o comercial.

Flexibilidad

Capa Física FreeDV

Construido sobre los modos de datos digitales FreeDV probados en campo, desarrollados por David Rowe (VK5DGR). El pool de modos elimina la reinicialización del codec — la configuración de enlace más rápida posible.

Tecnología Probada

GPL-3.0 Open Source

Código fuente completo. Sin tarifas de licencia. Sin dependencia de proveedor. Audita cada línea, haz fork, extiende, implementa. Desarrollo impulsado por la comunidad, patrocinado por ARDC.

Cómo Funciona

Operativo en Tres Pasos

Mercury está diseñado para la simplicidad sin sacrificar usabilidad ni rendimiento.

1
PASO 01

Conecta tu Radio HF

Conecta tu transceptor HF a la computadora mediante una interfaz de tarjeta de sonido (SignaLink, Rigblaster o incorporada). Mercury soporta cualquier radio que HAMLIB conozca — miles de modelos de Icom, Yaesu, Kenwood y más.

USB-C / Serial / Interfaz de Audio
2
PASO 02

Instala y Configura Mercury

Instala Mercury mediante un paquete Debian, compila desde el código fuente o descarga el ZIP para Windows. Edita un archivo de configuración INI simple para establecer tu indicativo, dispositivos de audio y modelo de radio. Eso es todo.

Linux, Windows, Raspberry Pi
3
PASO 03

Comienza a Comunicarte

Mercury expone una interfaz TCP compatible con VARA. Conecta Winlink Express, JS8Call o cualquier aplicación con capacidad ARQ. O usa Mercury directamente para enviar archivos, correo electrónico y datos a miles de kilómetros.

Winlink, JS8Call, Apps Personalizadas

Instala Mercury en Minutos

Elige tu plataforma y sigue los pasos. Documentación completa disponible en rhizomatica.github.io/mercury.

1 Instalar Dependencias
$ sudo apt-get update && sudo apt-get install \
libasound2-dev libpulse-dev libhamlib-dev make git
2 Clonar y Compilar
$ git clone https://github.com/Rhizomatica/mercury.git
$ cd mercury && make
$ sudo make install
3 Configurar y Ejecutar
$ cp mercury.ini.example mercury.ini
# Edita mercury.ini con tu indicativo y dispositivos de audio
$ mercury -z # Listar tarjetas de sonido disponibles
$ mercury -l # Listar modos de módem disponibles
$ mercury # Iniciar Mercury
1 Agregar Clave del Repositorio
$ wget --no-check-certificate -qO- \
https://debian.hermes.radio/hermes/hermes.key | \
gpg --dearmor -o - | \
sudo tee /etc/apt/trusted.gpg.d/hermes.gpg > /dev/null
2 Agregar Fuente (AMD64)
$ echo 'deb [arch=amd64] http://debian.hermes.radio/hermes trixie main' | \
sudo tee /etc/apt/sources.list.d/hermes.list
# Para ARM64 (Raspberry Pi), reemplaza amd64 con arm64
3 Instalar Mercury
$ sudo apt-get update
$ sudo apt-get install mercury
$ mercury --version # Verificar instalación
1 Descargar el ZIP
# 1. Visita la página de versiones:
# https://github.com/Rhizomatica/mercury/releases
# 2. Descarga el paquete ZIP más reciente para Windows
# 3. Extrae al directorio de tu preferencia
2 Configurar y Ejecutar
# Copia y edita el archivo de configuración:
$ copy mercury.ini.example mercury.ini
# Edita mercury.ini con el Bloc de Notas o VS Code
# Haz doble clic en mercury.exe para iniciar
3 Conectar Tu App
# Mercury expone un TNC TCP compatible con VARA:
# Control ARQ: localhost:8300
# Datos ARQ: localhost:8301
# Broadcast: localhost:8100
# Configura Winlink, JS8Call o tu app para conectar aquí
1 Requisitos Previos
# Requerido: GCC, Make, bibliotecas de desarrollo ALSA/PulseAudio, bibliotecas de desarrollo HAMLIB
$ sudo apt-get install build-essential libasound2-dev \
libpulse-dev libhamlib-dev
2 Clonar Repositorio
$ git clone https://github.com/Rhizomatica/mercury.git
$ cd mercury
# Revisa config.mk para opciones del compilador
$ cat config.mk
3 Compilar e Instalar
$ make -j$(nproc) # Compilar con todos los núcleos de la CPU
$ sudo make install # Instalar en todo el sistema
$ mercury -h # Ver todas las opciones
